# Changelog Page — Content Collections Data Source

## Context

Changelog page UI is mostly built (timeline, entry component, route, footer link). Currently uses a hardcoded array in `lib/changelog.ts`. Need to swap to Content Collections so entries are authored as MDX files — same pattern as the blog. This decouples changelog from GitHub releases (which are auto-tagged per PR, too granular).

## Content Collections Setup

Add a `changelog` collection in `content-collections.ts` alongside `posts`:

```ts
const changelog = defineCollection({
  name: "changelog",
  directory: "./src/content/changelog",
  include: "**/*.mdx",
  schema: z.object({
    title: z.string(),
    published: z.iso.date(),
    draft: z.boolean().default(false),
    category: z.enum(["data", "feature", "improvement", "community"]),
    summary: z.string(),
    content: z.string(),
  }),
  transform: async (document, context) => {
    const mdx = await compileMDX(context, document, {
      remarkPlugins: [remarkGfm],
    });
    return {
      ...document,
      slug: document._meta.path,
      mdx,
    };
  },
});
```

Register: `collections: [posts, changelog]`

## MDX File Format

```mdx
---
title: PLL 2025 Season Data
published: 2026-01-15
category: data
summary: Complete game-by-game statistics for the 2025 PLL season.
---

Body content here as markdown. Supports **bold**, lists, etc.

- 156 games across 8 teams
- Player stats: goals, assists, points, ground balls
```

Key difference from hardcoded model: `body[]` and `highlights[]` merge into a single MDX body. Simpler to author, richer formatting.

## Entry Component Adaptation

The `ChangelogEntryCard` currently renders `body[]` as paragraphs and `highlights[]` as bullet list. With MDX, the component renders compiled MDX directly using the existing `prose-blog` styles (or a lighter variant). The `summary` stays in frontmatter for the timeline preview text.

## Data Flow

```
src/content/changelog/*.mdx
  → content-collections build (bun run gen:content-collections)
  → allChangelogs (typed array)
  → changelog-posts.ts (filter drafts, sort by date)
  → route imports sorted array
  → ChangelogEntryCard renders each entry
```
